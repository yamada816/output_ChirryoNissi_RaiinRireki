# 治療日誌マスタ化出力ツール

治療日誌の `.xlsm` を複数読み込みし、`ThisWorkbook` 側に縦持ちマスタを生成する VBA ソースです。  
あわせて、店舗ごとの `新患分析 / id店舗名.xlsx` を参照し、患者名から顧客Noを引き当てます。

## 構成

- [src/modTreatmentDiaryImporter.bas](C:/Users/yuuki/OneDrive/ドキュメント/マイドキュメント/システム案件/治療日誌マスタ化出力ツール/src/modTreatmentDiaryImporter.bas)
- [dist/modTreatmentDiaryImporter_sjis.bas](C:/Users/yuuki/OneDrive/ドキュメント/マイドキュメント/システム案件/治療日誌マスタ化出力ツール/dist/modTreatmentDiaryImporter_sjis.bas)

## 前提

- 出力先はこの VBA を格納したブックです。
- 取り込み対象シートは `～10日` `11～20日` `21日～` の3枚です。
- 取り込み元ブックには `転記` シートがあり、`B35` に1ファイルの合計患者数が入っている前提です。
- 対象データは `A1:DF53` を固定範囲で読み取ります。
- ブロック開始列は `A, K, U, AE, AO, AY, BI, BS, CC, CM, CW` です。
- ファイル名の先頭数字を店舗IDとして扱います。例: `04学芸大.xlsm` -> `04`
- `店舗リファレンス` の `使用中フラグ` は空欄を有効扱いにしています。`0 / FALSE / NO / N / 無効 / 停止` は無効扱いです。
- 1回の取込は単一店舗運用を前提にしています。治療日誌と `id店舗名.xlsx` は同じ店舗のものを選択してください。
- `id店舗名.xlsx` は `新患分析` シートを持ち、`B列=顧客No` `C列=患者名` を使います。
- マクロを格納したブック名は `来院履歴マスタ_店舗ID_店舗名.xlsm` 形式を想定しています。例: `来院履歴マスタ_01_国立.xlsm`

## セットアップ

1. 任意のマクロ有効ブックを作成します。
2. VBA エディタで標準モジュールを追加し、[dist/modTreatmentDiaryImporter_sjis.bas](C:/Users/yuuki/OneDrive/ドキュメント/マイドキュメント/システム案件/治療日誌マスタ化出力ツール/dist/modTreatmentDiaryImporter_sjis.bas) をインポートします。
3. `InitializeTreatmentDiaryWorkbook` を実行します。
4. `InitializeTreatmentDiaryWorkbook` は、ブック名から `店舗ID / 店舗名` を取得して `店舗リファレンス` シートへ自動設定します。
5. ブック名から取得できなかった場合だけ、`店舗リファレンス` シートを手修正してください。

## 実行マクロ

- `InitializeTreatmentDiaryWorkbook`
  - 必要シートとヘッダーを作成します。
- `ImportTreatmentDiariesFromFolder`
  - 指定フォルダ配下の `.xlsm` を再帰的に取り込みます。
- `ImportTreatmentDiariesFromFiles`
  - 選択した複数 `.xlsm` を取り込みます。

どちらの取込マクロでも、取込開始時に `新患分析 / id店舗名.xlsx` の選択ダイアログが表示されます。

## 出力シート

- `治療日誌マスタ`
  - `顧客No, 顧客No判定状態, 内部取込ID, 日付, 年月, 施術時間, 分類, 担当, 患者名, 料金合計, 店舗ID, 店舗名, 元ファイル名, 元ファイルパス, 元シート名, 取込日時`
- `取込実行ログ`
  - `実行日時, ファイル名, シート名, ステータス, 読取件数, 出力件数, メッセージ`
- `取込詳細ログ`
  - `実行日時, ファイル名, シート名, 日付, 患者名, 担当, 種別, 内容`

## 顧客Noの引当ルール

- 患者名はスペースを除去した値で照合します。
- `新患分析` に同名が1件だけある場合は、その顧客Noを採用します。
- `新患分析` に同名が複数ある場合は、最大の顧客Noを最新として採用します。
- 同名が複数ある場合は詳細ログへ警告を出します。
- 一致なし、顧客No空欄、患者名空欄は仮値を採用します。
- 仮値は `-YYYYMMDDHHMMSSNNNN` 形式の一意な負値です。
- `顧客No判定状態` は `確定 / 警告 / 仮` のいずれかです。
- `内部取込ID` は `YYYYMMDDHHMMSSNNNN` 形式の取込内一意IDです。

## ログ出力

- 取込開始時に、前回までの `取込実行ログ` と `取込詳細ログ` を `.log` ファイルへ退避します。
- 退避後、シート上のログは2行目以降をクリアしてから今回の取込を開始します。
- ログファイルは、実行した `来院履歴マスタ_*.xlsm` と同じ階層の `log` フォルダへ出力します。
- 例: `C:\...\分析用データ\来院履歴マスタ_01_国立.xlsm` を実行した場合、`C:\...\分析用データ\log\import_YYYYMMDD_HHMMSS.log` へ出力します。
- `log` フォルダがなければ自動作成します。
- ログファイル名は `import_YYYYMMDD_HHMMSS.log` です。
- ブックのパスが `https://d.docs.live.net/...` のような OneDrive URL で取得される場合は、`FullName` からローカルの OneDrive 同期ファイルへ変換し、その親フォルダ直下へ `log` フォルダを作成します。
- 退避が実行された場合は、今回の `取込実行ログ` に `INFO` 行で退避先フルパスを残します。

## 実装済みルール

- `施術時間` が空でない行のみ取込対象です。
- `患者名` は前後空白を除去してマスタへ出力します。
- 顧客No照合時はさらにスペースを除去した患者名で比較します。
- 料金は `+4` から `+9` を合算します。
- 料金セルが文字列またはエラー値の場合は詳細ログに警告を残します。
- `担当空欄` と `患者名空欄` は警告ログを出しつつ処理継続します。
- `日付取得不可` `対象シートなし` `複数年月混在` `重複取込` `転記!B35不一致` はそのファイルを中止します。
- 再取込判定は論理ファイル名と `年月` で行います。

## エラー時の反映

- 取込データはファイル単位で検証し、最後にまとめて `治療日誌マスタ` へ出力します。
- そのため、通常のエラー時は該当ファイルのデータは `治療日誌マスタ` に反映されません。
- `重複取込` エラーだけは、すでに過去データが存在する前提なので、該当データを削除してから再実行してください。

## 警告メッセージ例

- 一致なし
  - `顧客ID（カルテNo）が取得できません。治療日誌の名前と新患分析の名前が一致していません。新患分析の名前を確認してください。治療日誌の名前: 山田太郎`
- 顧客No空欄
  - `顧客ID（カルテNo）が未登録です。新患分析の顧客Noを確認してください。治療日誌の名前: 山田太郎`
- 同名複数
  - `同じ名前の方がいます。候補: No101, No123。最新の顧客ID No123 を採用しました。`

## 取込時の注意

- VBA へ直接インポートする場合は Shift-JIS 版の [dist/modTreatmentDiaryImporter_sjis.bas](C:/Users/yuuki/OneDrive/ドキュメント/マイドキュメント/システム案件/治療日誌マスタ化出力ツール/dist/modTreatmentDiaryImporter_sjis.bas) を使ってください。
- UTF-8 版をそのままインポートすると、日本語のシート名やメッセージ定数が文字化けし、`インデックスが有効範囲にありません。` のような切り分けしづらいエラーになることがあります。
- `DELETE -> 再取込` 運用を前提にしており、既存マスタ行の更新機能は実装していません。
